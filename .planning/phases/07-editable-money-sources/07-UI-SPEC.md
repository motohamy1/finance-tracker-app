## Interaction Contracts

### MoneySourceCard: Tap-to-Edit Balance

1. User taps the balance amount text on a MoneySourceCard
2. Balance text fades out (150ms), TextInput with numeric keyboard fades in at same position
3. TextInput auto-focuses with current balance value pre-populated (formatted without currency symbol)
4. `keyboardType="decimal-pad"` — numeric keypad on both iOS and Android
5. Save triggers:
   - **Blur:** Tapping anywhere outside the TextInput saves and dismisses keyboard
   - **Enter/Return:** On Android hardware keyboards, Enter key saves
   - **Scroll:** Scrolling the card row blurs the input (FlatList keyboardShouldPersistTaps="handled")
6. On save: parse input, validate numeric, update store, animate back to display text
7. Invalid input (non-numeric): show subtle error (red border flash on card, 500ms), revert to previous value, do NOT show Alert
8. Empty input: treat as $0.00

### MoneySourceRow: Horizontal Scroll

1. `FlatList` with `horizontal={true}`, `showsHorizontalScrollIndicator={false}`
2. Auto-snap: `snapToInterval={cardWidth + gap}`, `decelerationRate="fast"`
3. Visual peeking: adjacent cards partially visible at edges (achieved via `contentInset` or padding at row boundaries)
4. Card reordering: long-press drag using existing drag-to-reorder pattern from Phase 1 categories (DraggableFlatList or gesture handler)
5. Loading state: skeleton placeholder cards (4 cards, 180px height, pulsing opacity) while data loads from SQLite

### MoneySourceCard: Long-Press Menu

1. Long-press on a MoneySourceCard opens action sheet (iOS) or Alert dialog (Android)
2. Options: "Edit Name", "Change Color", "Delete"
3. "Delete" triggers confirmation: see Copywriting Contract below
4. "Edit Name" opens inline rename (TextInput replaces name text, same pattern as balance editing)
5. "Change Color" opens a color palette selection sheet (grid of 8 color swatches from MONEY_SOURCE_PALETTE)

### ExpenseForm: Money Source Picker

1. Picker field appears between Category picker and Title input
2. Label: "Money Source (optional)"
3. Styled identically to category picker: `selectButton` style (backgroundColor #F8FAFC, borderRadius 10, padding 14, flexDirection row)
4. Selected state shows: colored dot (money source color), source name
5. Unselected state shows: "None" with gray dot
6. Dropdown lists all money sources with colored dots + "None" option at top
7. Context-based pre-select: if user tapped a specific money source card before opening form, that source is pre-selected (per D-10)

### TotalBalanceSummary

1. Displayed above MoneySourceRow in the `ListHeaderComponent`
2. Layout: compact row — label "Total Balance" (Label style, uppercase, letterSpacing 1) + amount (Display style, but smaller — 24px/600)
3. Amount = sum of all `moneySource.balanceCents` formatted as currency
4. Static display, no interaction
5. Updates reactively when any money source balance changes

### Money Source Creation/Edit Flow

1. "+" button at end of horizontal MoneySourceRow opens a creation sheet
2. Sheet fields: Name (required, TextInput), Color (grid picker, defaults to next palette color)
3. Creates money source with `balanceCents: 0`, `sortOrder: next`
4. New card animates in (slide from right, 300ms)
5. User taps balance ($0.00) to set initial amount

**Source:** Interactions derived from CONTEXT.md D-04 (horizontal scroll + peeking), D-05 (card content), D-14 (tap-to-edit), D-11 (picker in form), D-10 (context pre-select), D-15 ($0.00 default), D-16 (total sum). Agent discretion areas applied: auto-snap selected (not free-scroll), keyboard dismiss on blur, numeric keyboard type.
