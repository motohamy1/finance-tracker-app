## 2. Technical Spec

### FIFO Pairing Logic
- Matching algorithm: For each ticker, maintain a queue of "open" buy trades.
- When a sell trade occurs:
  1. Take the oldest buy trade (FIFO).
  2. Match shares. If buy shares > sell shares, split the buy. If sell shares > buy shares, match fully and move to the next buy in queue.
  3. Realized P&L = `(sellPrice - buyPrice) * matchedShares - (buyFeesRatio + sellFeesRatio)`.
  4. Fees are proportional if trades are split.

## 4. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| FIFO complexity with partial shares | High | Comprehensive unit tests for `pnl.ts` covering split buy/sell cases. |
| Floating point errors | Medium | Always use `cents` (integers) for all calculations. |
| Performance with large trade history | Low | Memoize P&L calculations in the store/selectors. |
