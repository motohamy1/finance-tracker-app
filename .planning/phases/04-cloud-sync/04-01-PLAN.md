    CREATE `src/services/auth.ts`:

    This service manages Google OAuth 2.0 authentication lifecycle using expo-auth-session with PKCE flow (required by Google for mobile apps).

    EXPORT these functions with exact signatures:

    ```typescript
    import * as AuthSession from 'expo-auth-session';
    import * as Crypto from 'expo-crypto';
    import * as SecureStore from 'expo-secure-store';
    import * as WebBrowser from 'expo-web-browser';

    // Google OAuth 2.0 endpoints (OpenID Connect discovery)
    const DISCOVERY = {
      authorizationEndpoint: 'https://accounts.google.com/o/oauth2/v2/auth',
      tokenEndpoint: 'https://oauth2.googleapis.com/token',
      revocationEndpoint: 'https://oauth2.googleapis.com/revoke',
    };

    // Scopes requested (D-01: Google Drive, D-02: appDataFolder)
    const SCOPES = [
      'https://www.googleapis.com/auth/drive.appdata',
      'https://www.googleapis.com/auth/userinfo.email',
    ];

    // SecureStore keys for token persistence
    const TOKEN_KEY = 'google_access_token';
    const REFRESH_KEY = 'google_refresh_token';
    const EMAIL_KEY = 'google_email';

    /**
     * Initiate Google Sign-In flow using PKCE (D-11).
     * Opens system browser for authentication.
     * Stores tokens in expo-secure-store on success.
     * Returns true if sign-in succeeded, false if cancelled or failed.
     */
    export async function signIn(): Promise<boolean>;

    /**
     * Sign out: revoke token via Google API, clear SecureStore,
     * and redirect to Google logout.
     */
    export async function signOut(): Promise<void>;

    /**
     * Get a valid access token. Attempts refresh if current token is expired.
     * Returns null if not authenticated or refresh fails.
     */
    export async function getAccessToken(): Promise<string | null>;

    /**
     * Returns true if a valid access token exists in SecureStore.
     * Does NOT validate token freshness — use getAccessToken for API calls.
     */
    export async function isAuthenticated(): Promise<boolean>;

    /**
     * Get the authenticated user's email from SecureStore.
     * Returns null if not signed in.
     */
    export async function getGoogleEmail(): Promise<string | null>;
    ```

    IMPLEMENTATION DETAILS:

    **signIn():**
    1. Call `WebBrowser.maybeCompleteAuthSession()` at module level (top of file, per expo-auth-session docs)
    2. Generate PKCE code verifier using `Crypto.getRandomBytes(32)` → base64url
    3. Create SHA-256 challenge from verifier using `Crypto.digestStringAsync(Crypto.CryptoDigestAlgorithm.SHA256, verifier)` → base64url
    4. Create `AuthRequest` with:
       - `clientId`: `process.env.EXPO_PUBLIC_GOOGLE_CLIENT_ID` (must be set in .env)
       - `scopes`: SCOPES array
       - `redirectUri`: `AuthSession.makeRedirectUri({ scheme: 'finance-tracker' })`
       - `usePKCE`: true
    5. Call `request.promptAsync(discovery)` where `discovery = { authorizationEndpoint: DISCOVERY.authorizationEndpoint }`
    6. If `response.type === 'success'`:
       - Exchange `response.params.code` for tokens at DISCOVERY.tokenEndpoint
       - Request body: `{ code, client_id, code_verifier, redirect_uri, grant_type: 'authorization_code' }`
       - Parse response: `{ access_token, refresh_token, expires_in, id_token }`
       - Decode id_token (JWT payload) to extract email
       - Store in SecureStore: access_token, refresh_token, email
       - Return true
    7. On any failure, return false

    **signOut():**
    1. Get access_token from SecureStore
    2. POST to `https://oauth2.googleapis.com/revoke?token={access_token}`
    3. Delete all three keys from SecureStore
    4. Call `WebBrowser.openAuthSessionAsync('https://accounts.google.com/logout')`
    5. Return void

    **getAccessToken():**
    1. Get access_token from SecureStore
    2. If missing, try refresh:
       - Get refresh_token from SecureStore
       - POST to DISCOVERY.tokenEndpoint with `{ refresh_token, client_id, grant_type: 'refresh_token' }`
       - Store new access_token
       - Return it
    3. If access_token exists, return it (executor will handle 401 on Drive API calls by calling this again after clearing cache)
    4. Return null if neither exists

    **isAuthenticated():**
    - Return `(await SecureStore.getItemAsync(TOKEN_KEY)) !== null`

    **getGoogleEmail():**
    - Return `await SecureStore.getItemAsync(EMAIL_KEY)`

    IMPORTANT: Use `process.env.EXPO_PUBLIC_GOOGLE_CLIENT_ID` for the client ID (NOT a hardcoded value). Document in code comments that the user must set this in `.env`:
    ```
    EXPO_PUBLIC_GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
    ```
    Also document in code comments that the user must configure the OAuth consent screen and redirect URI `finance-tracker://` in Google Cloud Console.

    NOTE ON user_setup: The executor will surface to the user that they need to create a Google Cloud OAuth 2.0 client ID with redirect URI `finance-tracker://` before this works.

    import { getAccessToken } from './auth';
    import type { SyncTable, SyncManifest } from '@/types';

    const DRIVE_API_BASE = 'https://www.googleapis.com/drive/v3';
    const DRIVE_UPLOAD_BASE = 'https://www.googleapis.com/upload/drive/v3';
    const SYNC_FOLDER = 'sync';  // Virtual folder prefix within appDataFolder

    /** File ID for the sync manifest — always named "manifest.json" */
    const MANIFEST_NAME = `${SYNC_FOLDER}/manifest.json`;

    /**
     * List all sync JSON files in appDataFolder.
     * Returns array of { name: "sync/categories.json", id: "drive-file-id", modifiedTime: "ISO" }
     */
    export async function listSyncFiles(): Promise<Array<{ name: string; id: string; modifiedTime: string }>>;

    /**
     * Upload JSON data to a file in appDataFolder.
     * @param tableName — e.g., "categories" → stored as "sync/categories.json"
     * @param data — the JSON-serializable array of records
     * @param existingFileId — if updating an existing file, pass its Drive file ID (null for new files)
     * @returns the Drive file ID
     */
    export async function uploadJSON(tableName: SyncTable, data: unknown[], existingFileId?: string | null): Promise<string>;

    /**
     * Download and parse a JSON file from appDataFolder.
     * @param fileId — Drive file ID (from listSyncFiles)
     * @returns parsed JSON array
     */
    export async function downloadJSON<T = unknown>(fileId: string): Promise<T[]>;

    /**
     * Upload the sync manifest file.
     * @param manifest — SyncManifest object
     * @param existingFileId — Drive file ID of existing manifest (null for first upload)
     */
    export async function uploadManifest(manifest: SyncManifest, existingFileId?: string | null): Promise<string>;

    /**
     * Download and parse the sync manifest.
     * @returns SyncManifest or null if no manifest exists
     */

    /**
     * Delete a file from appDataFolder by Drive file ID.
     */
    export async function deleteFile(fileId: string): Promise<void>;

    IMPLEMENTATION DETAILS:

    **Helper — makeAuthenticatedRequest(method, url, body?, contentType?):**
    1. Get token via `await getAccessToken()`
    2. If token is null, throw `new Error('Not authenticated — call signIn() first')`
    3. Make fetch call with headers:
       ```
       Authorization: Bearer {token}
       Content-Type: {contentType || 'application/json'}
       ```
    4. If response.status === 401:
       - Clear cached token in auth service (import and call a token invalidation function, or handle internally)
       - Try getAccessToken() once more (triggers refresh)
       - Retry the request
       - If still 401, throw authentication error
    5. If !response.ok, throw with response body text
    6. Return parsed JSON or raw response as appropriate

    **listSyncFiles():**
    - GET `{DRIVE_API_BASE}/files?spaces=appDataFolder&fields=files(id,name,modifiedTime)&q=name contains 'sync/'`
    - Return array of `{ name, id, modifiedTime }`

    **uploadJSON(tableName, data, existingFileId):**
    - File name: `sync/{tableName}.json` per D-06 granular sync
    - Use multipart upload for new files:
      1. POST `{DRIVE_UPLOAD_BASE}/files?uploadType=multipart`
      2. Body: multipart with metadata part + JSON content part
      3. Metadata: `{ name: "sync/{tableName}.json", parents: ["appDataFolder"] }`
      4. Content: `JSON.stringify(data)` with Content-Type: application/json
    - Use PATCH `{DRIVE_API_BASE}/files/{existingFileId}?uploadType=media` to update existing
    - Return file.id from response

    **downloadJSON<T>(fileId):**
    - GET `{DRIVE_API_BASE}/files/{fileId}?alt=media`
    - Parse response as JSON
    - Return as T[]

    **uploadManifest(manifest, existingFileId):**
    - Same as uploadJSON but uses MANIFEST_NAME as filename
    - Type is not SyncTable — special case

    **downloadManifest():**
    1. Call listSyncFiles()
    2. Find file where name === 'sync/manifest.json'
    3. If found, call downloadJSON with its id
    4. Cast to SyncManifest
    5. If not found, return null

    **deleteFile(fileId):**
    - DELETE `{DRIVE_API_BASE}/files/{fileId}`
    - Ignore 404 (already deleted)

    FILE NAMING CONVENTION (D-03, D-06):
    - `sync/categories.json` — Category records array
    - `sync/expenses.json` — Expense records array
    - `sync/trades.json` — Trade records array
    - `sync/current_prices.json` — CurrentPrice records array
    - `sync/manifest.json` — SyncManifest metadata

    NOTE: The Drive appDataFolder has a hidden quota — the user cannot see these files in their Drive UI. They can only be accessed via the API with the app's credentials.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App → Google OAuth | User credentials and authorization code flow via system browser |
| App → Google Drive API | Access token sent in Authorization header; JSON data in transit |
| SecureStore → App | Tokens loaded from device keychain/keystore at runtime |
| User → Settings UI | Tapping sign-in/sign-out buttons triggers auth flow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | `auth.ts` — Google Sign-In | mitigate | Use PKCE flow (expo-auth-session built-in) to prevent authorization code interception. Validate id_token nonce to prevent replay. |
| T-04-02 | Information Disclosure | `auth.ts` — Token storage | mitigate | Store access/refresh tokens in expo-secure-store (iOS Keychain / Android EncryptedSharedPreferences), never in AsyncStorage or plain filesystem. Never log tokens to console. |
| T-04-03 | Tampering | `drive.ts` — Data transfer | mitigate | Google Drive API uses HTTPS (TLS 1.3 in transit). appDataFolder scope ensures no other app can read/write these files. Files are private to the app. |
| T-04-04 | Information Disclosure | `drive.ts` — API calls | mitigate | Use Authorization: Bearer header over HTTPS. Never include access token in URL query parameters. |
| T-04-05 | Elevation of Privilege | `auth.ts` — Client ID config | mitigate | Use process.env.EXPO_PUBLIC_GOOGLE_CLIENT_ID (environment variable, not hardcoded). The OAuth client ID is scoped to `drive.appdata` only — cannot access user's general Drive files. |
| T-04-06 | Denial of Service | `drive.ts` — Rate limiting | accept | Google Drive API has quota limits (10,000 requests/day free tier). Single-user app with infrequent sync is unlikely to hit limits. No PII at risk if rate-limited. |
</threat_model>
